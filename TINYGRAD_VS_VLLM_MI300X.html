Verdict

vLLM has 14 MI300X optimizations that tinygrad lacks.

The critical gaps are:

  1. AITER library — vLLM's secret weapon. Provides optimized Flash Attention, MLA, fused MoE, fused RMSNorm, fused RoPE+cache, and grouped_topk kernels specifically for MI300X. Tinygrad has nothing comparable.
  2. PagedAttention (ll4mi) — A hand-tuned LDS-based decode kernel for MI300X. Essential for LLM serving.
  3. hipBLASLt online tuning — vLLM enables runtime autotuning of GEMM operations. Tinygrad uses generic COMGR compilation.
  4. KV cache shuffle layout — LDS-friendly memory layout for attention kernels. Tinygrad doesn't have this.
  5. CUDA/HIP Graph capture — Reduces CPU-side launch overhead. Critical for decode throughput.

Tinygrad has 4 advantages:

  1. Direct AM driver — Bypasses entire ROCm stack for lower latency and smaller footprint.
  2. BEAM search — Can discover better kernel configurations at runtime.
  3. Explicit XCC dispatch — Direct control over multi-XCC scheduling.
  4. Scratch swizzle — Better private memory access patterns.

Bottom line: vLLM is optimized for LLM serving on MI300X (attention + GEMM + quantization + fused kernels). Tinygrad is a general-purpose tensor compiler that can target MI300X but lacks the application-specific optimizations (paged attention, flash attention, fused kernels) that make vLLM fast for LLM inference. Tinygrad's strengths are in lower-level control (AM driver, BEAM search) that vLLM doesn't have.
